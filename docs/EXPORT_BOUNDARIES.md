# Export Boundaries

Status: Sprint 01.5 remediation decision

## Decision

Core Imprint exports profile contracts, observed expression patterns, constraints, evidence
metadata, confidence, sample policy, and warnings.

Core Imprint does not export generation-ready prompts as its source of truth. Downstream adapters
may translate profile contracts into prompts or product-specific instructions.

## What Imprint Can Export

Allowed core exports:

- canonical Imprint JSON
- canonical Imprint YAML
- human-readable report
- expression profile fragment for another system
- source summary and confidence metadata
- anti-pattern guidance
- platform/context fit notes
- warnings and prohibited-use metadata
- public-safe, private-local, and full-local export variants

These exports describe what the profile says. They do not own how a target model should be
prompted in a specific application.

## What Downstream Systems Own

Downstream systems own:

- draft generation
- prompt assembly
- system prompts
- model-specific instruction tuning
- decoding and sampling settings
- publishing workflow
- brand enforcement
- persona simulation
- conversation behavior
- approval and editorial review

## Prompt Fragment Policy

The phrase "publishing prompt contract exporter" should be retired from core architecture.

If prompt-like material exists, it must be:

- generated by a downstream adapter or integration layer
- derived from canonical profile fields
- labeled as consumer-specific
- excluded from canonical schema semantics
- tested as a projection, not as the profile source of truth

## Export Review Gate

Every new exporter must pass this review question:

> Does this exporter decide how a model should generate text in a specific downstream workflow?

If yes, it is an adapter concern, not a core Imprint exporter.

Allowed core exporter behavior:

- emit structured profile fields
- emit constraints and anti-patterns
- emit evidence and confidence metadata
- emit target-consumer identifiers

Disallowed core exporter behavior:

- assemble system prompts
- tune prompts for a provider or model
- encode editorial workflow steps
- decide sampling or decoding behavior
- generate drafts

## Rejected Alternatives

### Core Prompt Exporter

Rejected because it makes Imprint responsible for generation behavior and downstream model
context.

### No Downstream Guidance

Rejected because profiles need to be useful. Imprint can export structured constraints and
patterns; consumers decide how to apply them.

### Consumer-Specific Profiles as Source of Truth

Rejected because it fragments the contract and makes migrations unsafe.

## Sprint 02 Implications

Sprint 02 schemas should define:

- canonical export contract
- export mode
- target consumer metadata
- projection metadata
- no raw examples by default
- no generation workflow ownership in core profile objects
