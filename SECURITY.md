# Security Policy

Imprint processes sensitive personal and professional artifacts in private deployments. Treat privacy and security issues as high priority.

## Supported versions

Imprint is pre-release. There are no stable supported versions yet.

## Reporting a vulnerability

If you find a security or privacy issue, please do not publish sensitive details in a public issue.

Open a minimal public issue asking for maintainer contact, or contact the maintainer through the project owner's published GitHub profile.

## Sensitive data policy

Do not include private corpora, messages, emails, transcripts, profile exports, API keys, database dumps, hostnames, or screenshots containing private information in public issues or pull requests.

## Design stance

- Local-first operation should be possible.
- Remote providers must be explicit.
- Public examples must be synthetic.
- Raw private research belongs under ignored `private/` paths.
- Public-safe exports must not include raw private examples by default.
